About DNS

VintageHive includes a DNS server that intercepts all domain name lookups and resolves them to the VintageHive server's IP address. This allows vintage computers to connect to VintageHive services by hostname without needing a real DNS server or hosts file entries.

How it works: Every A record query (hostname → IP address) returns VintageHive's IP. The vintage computer then connects to VintageHive, which proxies the request to the appropriate service (web proxy, FTP, IRC, etc.).

Non-standard port: VintageHive's DNS server listens on port {{ port_dns }} by default instead of the standard port 53. Some operating systems allow you to specify a custom DNS port, while others require port 53. If your OS requires port 53, you can change VintageHive's DNS port in the admin panel, or set up port forwarding.

Note: AAAA (IPv6), MX, CNAME, and all other query types return an empty response (NOERROR with zero answers). Only A record lookups are needed for vintage networking.

Windows 95 / 98 / Me

Setting DNS in TCP/IP Properties:

  1. Open Control Panel > Network
  2. Select TCP/IP bound to your network adapter and click Properties
  3. Click the DNS Configuration tab
  4. Select Enable DNS
  5. Under DNS Server Search Order, add:
    {{ serverip }}
  6. Click OK, then OK again. Restart if prompted.

Windows 9x uses standard port 53 for DNS. If VintageHive's DNS port is not 53, you will need to configure port forwarding on your network, or change the DNS port to 53 in VintageHive's admin panel.

Windows NT 4.0 / 2000 / XP

Windows 2000 / XP:

  1. Open Control Panel > Network Connections
  2. Right-click your connection, select Properties
  3. Select Internet Protocol (TCP/IP) and click Properties
  4. Select Use the following DNS server addresses
  5. Set Preferred DNS server to:
    {{ serverip }}
  6. Click OK and close the dialogs

Windows NT 4.0:

  1. Open Control Panel > Network
  2. Click the Protocols tab, select TCP/IP Protocol, click Properties
  3. Click the DNS tab
  4. Under DNS Service Search Order, click Add
  5. Enter {{ serverip }} and click Add
  6. Click OK and restart if prompted

Like Windows 9x, these versions use port 53 only.

Mac OS

Mac OS 7.x / 8.x / 9.x (Open Transport):

  1. Open Control Panels > TCP/IP
  2. Set Connect via to Ethernet
  3. Set Configure to Manually (or keep DHCP and override the Name Server)
  4. In the Name server addr. field, enter:
    {{ serverip }}
  5. Close the control panel and save changes

Mac OS X:

  1. Open System Preferences > Network
  2. Select your connection, click Advanced
  3. Click the DNS tab
  4. Click + and add {{ serverip }}
  5. Remove any other DNS servers
  6. Click OK, then Apply

Unix / Linux

Edit /etc/resolv.conf:

$ sudo vi /etc/resolv.conf

nameserver {{ serverip }}

Replace any existing nameserver lines with VintageHive's IP address. On some Linux distributions, NetworkManager or systemd-resolved may overwrite this file. Check your distribution's documentation for making DNS changes permanent.

Standard resolv.conf uses port 53 only. If VintageHive's DNS port is non-standard, use port forwarding (e.g., iptables) or configure the DNS port to 53 in VintageHive.

Testing Your DNS Configuration

Using nslookup (Windows / Unix):

$ nslookup www.yahoo.com {{ serverip }}

Server: {{ serverip }}
Address: {{ serverip }}#{{ port_dns }}

Name: www.yahoo.com
Address: {{ serverip }}

Using dig (Unix / Mac OS X):

$ dig @{{ serverip }} -p {{ port_dns }} www.yahoo.com A

;; ANSWER SECTION:
www.yahoo.com. 300 IN A {{ serverip }}

If both commands return VintageHive's IP address, DNS is working correctly. All domain names should resolve to the same IP — this is by design, as VintageHive intercepts all traffic and routes it through its proxy services.

Printing from DOS

DOS applications print to LPT ports. Redirect LPT1 to VintageHive's Raw TCP port:

C:\> net use LPT1: \\{{ serverip }}\PRINTER

If your DOS TCP/IP stack supports LPR, you can also use:

C:\> lpr -S {{ serverip }} -P printer filename.txt

Most DOS applications (WordPerfect, Lotus 1-2-3, etc.) will print using ESC/P or IBM ProPrinter commands. Select an Epson FX-80 or IBM ProPrinter driver in your application for best results.

Printing from Windows 3.1x

Windows 3.1x can print via LPT port redirection (same as DOS above), or through a network redirector if your TCP/IP stack supports it.

  1. In Control Panel, open Printers
  2. Click Add and select a PostScript or ESC/P compatible driver (e.g. Apple LaserWriter for PostScript, Epson FX-80 for dot matrix)
  3. Set the port to LPT1:
  4. From a DOS prompt, redirect LPT1 to VintageHive:
C:\> net use LPT1: \\{{ serverip }}\PRINTER

Printing from Windows 95/98/ME

Option A: Raw TCP (Recommended)

  1. Open Start > Settings > Printers
  2. Run Add Printer wizard, choose Network printer
  3. For the network path, enter: \\{{ serverip }}\PRINTER
  4. Select a printer driver — HP LaserJet 4 (PCL) or Apple LaserWriter (PostScript) work well

Option B: LPR (requires Microsoft LPR client)

  1. Install Microsoft TCP/IP Printing from Windows Setup if not already present
  2. In Add Printer, select Network printer
  3. Choose LPR Port, enter server: {{ serverip }}, queue: printer
  4. Select your printer driver and finish the wizard

Printing from Windows 2000/XP

Option A: IPP (Recommended)

  1. Open Printers and Faxes (or Printers)
  2. Run Add Printer, select Network printer
  3. Choose Connect to a printer on the Internet
  4. Enter URL: http://{{ serverip }}:{{ port_ipp }}/ipp/print
  5. Select a PostScript driver (e.g. HP Color LaserJet PS)

Option B: LPR Port

  1. In Add Printer, choose Local printer (uncheck auto-detect)
  2. Create a new port > Standard TCP/IP Port
  3. Enter address: {{ serverip }}
  4. In port settings, select LPR protocol, queue name: printer
  5. Select your printer driver and finish

Option C: Raw TCP (Port 9100)

  1. Same as LPR above, but in port settings select Raw protocol
  2. Port number: {{ port_rawprint }}

Printing from Mac OS Classic (7.x - 9.x)

Mac OS Classic can print via LPR using the Desktop Printer Utility or LaserWriter 8 driver:

  1. Open Chooser from the Apple menu
  2. Select LaserWriter 8
  3. If LPR printers are not shown, use the Desktop Printer Utility:
  4. Choose Printer (LPR), click Change...
  5. Enter LPR address: {{ serverip }}, queue: printer
  6. Select a PostScript Printer Description (PPD) file — Generic or any LaserWriter PPD

Printing from Mac OS X

Option A: IPP (Recommended)

  1. Open System Preferences > Print & Fax
  2. Click + to add a printer
  3. Select the IP tab
  4. Protocol: Internet Printing Protocol - IPP
  5. Address: {{ serverip }}
  6. Queue: ipp/print
  7. Select Generic PostScript Printer as the driver

Option B: LPD

  1. Same steps as above, but select Protocol: Line Printer Daemon - LPD
  2. Address: {{ serverip }}
  3. Queue: printer

Printing from Linux/Unix

Using CUPS (IPP):

$ lpadmin -p VintageHive -E \
  -v ipp://{{ serverip }}:{{ port_ipp }}/ipp/print \
  -m everywhere

Using lpr (LPD):

$ lpr -P printer@{{ serverip }} filename.ps

Using netcat (Raw TCP):

$ cat filename.ps | nc {{ serverip }} {{ port_rawprint }}

Printer Driver Recommendations
Application Type Recommended Driver Output Format
DOS text / word processors Epson FX-80 / IBM ProPrinter ESC/P
Windows GUI applications Apple LaserWriter / HP LaserJet PS PostScript
Mac applications LaserWriter 8 / Generic PostScript PostScript
Plain text files Generic / Text Only Plain text

VintageHive automatically detects the print data format (PostScript, ESC/P, IBM ProPrinter, PCL, or plain text) and converts it to PDF.

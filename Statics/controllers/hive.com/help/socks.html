About SOCKS

VintageHive includes a SOCKS proxy that handles SOCKS4, SOCKS4a, and SOCKS5 connections on a single port. SOCKS is a general-purpose proxy protocol that works at the TCP level — any application that supports SOCKS can tunnel its connections through VintageHive.

Unlike an HTTP proxy (which only handles web traffic), SOCKS can proxy any TCP connection: FTP clients, IRC clients, Telnet sessions, and more. Many vintage applications that don't have built-in HTTP proxy support do support SOCKS.

Which version should I use?

Version Features
SOCKS4 Basic TCP connect. Client resolves hostnames. IPv4 only.
SOCKS4a Like SOCKS4, but the server resolves hostnames (better for vintage clients without DNS).
SOCKS5 TCP connect with server-side DNS, IPv4 and IPv6 support.

If your application lets you choose, SOCKS5 is recommended. If it only supports SOCKS4, enable the "4a" option if available so the server handles DNS resolution.

Internet Explorer

Internet Explorer 4.x / 5.x / 6.x:

  1. Open Tools > Internet Options
  2. Click the Connections tab, then LAN Settings
  3. Check Use a proxy server
  4. Click Advanced
  5. In the Socks row, fill in:
    Field Value
    Socks {{ serverip }}
    Port {{ port_socks }}
  6. Leave the HTTP, Secure, and FTP rows empty (unless you also want VintageHive's HTTP proxy — see Client Settings)
  7. Click OK three times to save

Note: Internet Explorer uses SOCKS4 by default. SOCKS5 requires a registry tweak or using a third-party SOCKS client like SocksCap.

Netscape Navigator

Netscape Communicator 4.x:

  1. Open Edit > Preferences
  2. Navigate to Advanced > Proxies
  3. Select Manual proxy configuration and click View
  4. In the SOCKS Host field, enter:
    Field Value
    SOCKS Host {{ serverip }}
    Port {{ port_socks }}
  5. Click OK

Netscape Navigator 3.x:

  1. Open Options > Network Preferences
  2. Click the Proxies tab
  3. Select Manual Proxy Configuration and click View
  4. Set SOCKS Host to: {{ serverip }}
  5. Set Port to: {{ port_socks }}
  6. Click OK

Netscape 4.x supports both SOCKS4 and SOCKS5 (set via the SOCKS version radio buttons in the proxy dialog).

Trumpet Winsock / SocksCap

Trumpet Winsock 3.x:

Trumpet Winsock can route all TCP connections through a SOCKS server, making it ideal for 16-bit Windows applications that have no proxy support of their own.

  1. Open File > Setup
  2. Click the SOCKS tab
  3. Check Enable SOCKS Firewall
  4. Set Server to: {{ serverip }}
  5. Set Port to: {{ port_socks }}
  6. Click OK

SocksCap / SocksCap32:

SocksCap wraps any Winsock application to use SOCKS5. This is useful for programs that don't have built-in SOCKS support.

  1. Open SocksCap and go to File > Settings
  2. Set SOCKS Server to: {{ serverip }}
  3. Set Port to: {{ port_socks }}
  4. Set Protocol to: SOCKS5
  5. Leave Authentication as No Authentication
  6. Click OK
  7. Drag application shortcuts into SocksCap, then launch them from there

Command Line (curl)

Testing with curl:

$ curl --socks5 {{ serverip }}:{{ port_socks }} http://example.com

$ curl --socks4 {{ serverip }}:{{ port_socks }} http://example.com

$ curl --socks4a {{ serverip }}:{{ port_socks }} http://example.com

SSH tunnel via SOCKS:

$ ssh -o ProxyCommand="nc -X 5 -x {{ serverip }}:{{ port_socks }} %h %p" user@host

Application-Level SOCKS

Many vintage applications have built-in SOCKS support. Here are some common ones:

Application Where to Configure
mIRC File > Options > Connect > Firewall. Set to SOCKS5, enter server and port.
CuteFTP Edit > Settings > Connection > SOCKS. Set type, server, and port.
WS_FTP Session Properties > Firewall tab. Set type to SOCKS, enter server and port.
Eudora Tools > Options > Advanced Network. Set SOCKS server and port.
Forte Agent Options > Firewall. Set type to SOCKS5, enter server and port.

For applications without SOCKS support, use SocksCap (Windows) or tsocks (Unix) to transparently redirect their connections through the SOCKS proxy.

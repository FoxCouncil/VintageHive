Internet Archive (Wayback Machine)

VintageHive uses the Internet Archive Wayback Machine to serve archived versions of websites. When you visit a URL through the proxy, VintageHive looks up that page in the Wayback Machine and returns the archived version.

How it works:

  1. Configure your browser's proxy settings (see Client Settings)
  2. Browse to any website URL as you normally would
  3. VintageHive checks ProtoWeb first, then falls back to the Internet Archive
  4. The archived page is cleaned up and served to your vintage browser

Target Year: By default, VintageHive fetches pages from 1999. The target year can be changed (1997–2021) through the Admin Panel at http://admin.hive.com.

Caching: Archived pages are cached locally for 365 days to improve loading speed. Caches can be cleared through the Admin Panel.

Note: Not every page on the internet was archived. If a page wasn't captured by the Wayback Machine in your target year, you may see a not-found page.

ProtoWeb

ProtoWeb is a community project that maintains high-quality archives of classic websites. VintageHive checks ProtoWeb before the Internet Archive, so ProtoWeb sites load with better fidelity and accuracy.

How it works:

  • VintageHive maintains a list of all sites available on ProtoWeb
  • When you visit a URL, it checks if that domain exists on ProtoWeb
  • If found, the page is served from ProtoWeb's archive
  • If not found, it falls through to the Internet Archive

Browsing ProtoWeb sites:

You can browse the full catalog of available ProtoWeb sites from the VintageHive intranet portal at http://hive.com under the ProtoWeb Browser section. This includes both HTTP websites and FTP archives.

Direct passthrough sites:

Some retro-friendly websites are served live from the modern web (not archived). These include:

  • 68k.news — news formatted for vintage browsers
  • frogfind.com — a search engine for old browsers
  • razorback95.com — Windows 95/98 software archive
  • webtv.zone — WebTV community site
  • windowsupdaterestored.com — classic Windows Update

These sites are fetched directly and do not go through the Wayback Machine or ProtoWeb.

DialNine (HTTPS Proxy)

DialNine is VintageHive's HTTPS/SSL proxy. It allows vintage browsers to access HTTPS websites that would otherwise be unreachable due to modern TLS requirements.

How it works:

  • Your browser sends the HTTPS request to VintageHive's HTTPS proxy port
  • VintageHive connects to the target site using modern TLS on your behalf
  • The content is relayed back to your browser over a legacy SSL connection that older browsers can understand

Setup:

  1. In your browser's proxy settings, set the Secure / HTTPS proxy to: {{ serverip }}:{{ port_https }}
  2. Install VintageHive's CA certificate (see Client Settings > Compatibility Tips)
  3. Browse HTTPS sites normally — the proxy handles the rest

Certificate Installation:

  1. Visit http://admin.hive.com/ca.crt in your browser
  2. Install the certificate when prompted (see Client Settings for per-browser instructions)

Note: HTTPS support is experimental. Some sites may not render correctly, and certificate warnings may appear in some browsers. Not all vintage browsers support SSL.

Browser Tips

Recommended Browsers:

Browser Best For
Internet Explorer 5.x / 6.x Best overall compatibility; supports CSS1, JavaScript, SSL
Netscape Navigator 4.x Good compatibility; full SSL with cert import
Internet Explorer 3.x Works well for HTML-only sites; limited CSS
NCSA Mosaic Basic HTML browsing; no SSL, limited features
Opera 3.x–5.x Fast and lightweight; good CSS support

Browsing Tips:

  • Start at the portal: Visit http://hive.com for news, weather, web search, and curated links
  • Search the web: Use the search bar on http://hive.com — it uses DuckDuckGo and returns results optimized for vintage browsers
  • Article reader: VintageHive includes a built-in article reader that strips modern web clutter and renders clean HTML
  • Change the year: If a site looks wrong, try changing the Internet Archive target year through the Admin Panel
  • JavaScript: Most archived sites work fine without JavaScript. If a page is broken, it may rely on scripts that can't be archived
  • Images: Large modern images may load slowly on vintage hardware. Some browsers support disabling images for faster browsing
